# commit.md

Коммиты всегда содержат детальное описание после заголовка с планом, выполненными шагами и результатами для лучшей читаемости истории.  

## Обязательная структура

```
<type>: <описание>

План:
- Задача 1
- Задача 2  
- Задача 3

Выполнено:
- Реализовано 1
- Реализовано 2

Результат: 
Итог работы, готовность к ревью
```

## Правила оформления

- **Заголовок**: `<type>: <описание>` (типы: `feat`, `fix`, `refactor`, `docs`, `style`, `test`, `chore`, `WIP`) 
- **План**: 2-4 пункта в инфинитиве 
- **Выполнено**: список реализованного 
- **Результат**: итог и статус 
- Русский язык, повелительное наклонение в заголовке
